一、方法与消息

SEL

SEL又叫方法选择器，表示一个方法，是指向objc_selector结构体的指针。

Objective-C在编译时会根据方法名和参数序列生成一个整型的标识，这个标识就是SEL。只要方法名相同，那么SEL就相同，所以在同一个类或者类的继承体系中不能存在多个相同的方法名。

Objective-C中有三种方法可以获得SEL：

IMP

IMP是一个函数指针，指向的是方法的实现地址。

其中第一个参数是指向self（如果是实例方法，指向的是实例的内存地址；如果是类方法，指向的是该类的元类）；第二个参数是方法的SEL；以后的参数是方法的参数列表。
所以根据方法的SEL就可以获取IMP，取得IMP后就可以得到方法的入口，这样就可以使用这个函数指针了。

方法调用流程

在我们调用方法时[testObject testMethod]，编译器会将其转变为一段代码objc_msgSend(testObject, selector)其中第一个参数为调用方法的对象，第二个参数为方法的selector。
首先objc_msgSend通过testObject的isa指针找到其所属类的结构体，在类的方法分发表中查找与selector相同的方法，如果未找到则在其父类中查找。找到方法后根据IMP指针进入方法入口执行方法，同时将方法存储在cache中以便下次更快的调用方法；如果找不到方法，就会走消息转发流程。

消息动态处理

在消息未处理时，会调用所属类的方法：
+ (BOOL)resolveInstanceMethod:(SEL)sel（实例方法）
+ (BOOL)resolveClassMethod:(SEL)sel（类方法）
在这个方法中我们可以利用动态特性处理是否处理方法。

备用接收者

如果上一步无法处理，则由forwardingTargetForSelector来处理，如果一个实例实现了这个方法并返回非nil，则Runtime将这个消息转发给这个对象(但是不能是self，不然会出现死循环)，这一步只能进行消息的转发，不能对消息的参数等进行修改。

消息转发

如果上述方法都无法处理消息，就会执行正式的消息转发流程。
消息转发主要有两个方法：
- (NSMethodSignature *)methodSignatureForSelector:(SEL)aSelector
- (void)forwardInvocation:(NSInvocation *)anInvocation
